Sistemas de Informação - UFJF

- Descrição do sistema:

O sistema será desenvolvido para que possa controlar o fluxo nas mesas de um bar. E deverá seguir as seguintes expecificações:

  • Mesas e produtos deverão ser previamente cadastrados.
  • O atendente irá abrir uma mesa, colocando a data, o nome do responsavel pela mesa assim como 1 produto.
  • A mesa poderá ser alterada a qualquer momento, inserindo novos produtos.
  • No fechamento da mesa será exibido o total do pedido, assim como o valor dividido, caso o cliente solicite.
  • Inicialmente, não será feita a impressão do pedido, somente a vizualização do fechamento na tela.

- Tipo de dados utilizados:

Os dados utilizados no sistema são por meio de arquivos .txt, com isso, conseguimos manter a persistência dos dados. Foram criados 4 arquivos, mesas.txt, produtos.txt, pedidos.txt e movi_pedidos.txt, no momento da abertura de cada tela, esses arquivos são lidos e inseridos cada linha nos arrays correspondentes a cada arquivo, após cada gravação, o arquivo é excluido e o array que guarda os dados é gravado novamente no arquivo correspondente.

- Dificuldades encontradas:

As maiores dificuldades encontradas foi na montagem do layout das telas, fazendo tudo "na mão" não da pra posicionar o elemento exatamente onde se deseja.

- Melhorias futuras:

Arquivos .txt são uma ótima alternativa para soluções pequenas e rápidas, porém, para escrita, é necessário apagar o arquivo, e reescrever todos os registros. Quando se tem um sistema que serão gerados muitos resgistros ao longo do tempo, acaba ficando muito lento. A melhor alternativa é a implementação de banco de dados, onde se tem uma velocidade maior no acesso aos dados.
